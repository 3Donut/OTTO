3 metastuff::registerMembers<T> is used for class registration and it has the following form when
4 specialized:
5 
6 namespace metastuff
7 {
8 
9 template <>
10 auto registerMembers<YourClass>()
11 {
12  return members(
13  member(...),
14  ...
15  );
16 }
17 
18 }
19 
20 ! Some important details:
21 1) This specialization should be defined in header, because compiler needs to deduce the return
22 type. 2) This function is called by MetaHolder during it's static member initialization, so the
23 tuple is created only once and then registerMembers function is never called again. 3)
24 registerMembers could easily be a free function, but befriending such function is hard if you want
25 to be able to get pointers to private members... Writing "friend class Meta" in your preferred class
26  is just much easier. Though this might be somehow fixed in the future.
27 4) If the class is not registered then doForAllMembers(<your function>) will do nothing,
28  because the function will return empty tuple.
29 5) MemberPtr.h is included in this file just so that user can #include "Meta.h" and get MemberPtr.h
30  included too, which is always needed for registration.
31
